단순히 예외, 혹은 에러가 발생하지 않도록 하는 것 뿐만 아니라 에러가 명확하게 발생하도록 처리해야 함.
Spring RuntimeException의 경우 status code는 500임. 그런데 클라이언트에서 서버에 존재하지 않는 데이터를 찾으려고 하는 경우에도 서버쪽 코드에서 RuntimeException으로 처리하면 500이 떨어진다. 하지만 이건 400번대 에러가 더 적절함. 이런 부분에 대한 처리가 필요하다.

## HTTP Status Code
1. Informational responses (100 – 199)
2. Successful responses (200 – 299)
3. Redirection messages (300 – 399)
4. Client error responses (400 – 499)
5. Server error responses (500 – 599)
